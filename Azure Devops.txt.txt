What are artifacts?
---
A WAR file is basically a ZIP file that contains your entire Java web application. Everything required to run your web application. Artifacts are what you deploy to servers like Tomcat, JBoss, WebLogic, Azure App Service, etc.

Which deployment strategy are you using in your organization?
---
# There are 2 types of developments 1) Blue Green Deployment 2) Canary Deployment

---> Blue‑Green Deployment Strategy is a deployment method used to release new versions of applications with zero downtime and minimal risk.
 
* Blue – the current live/production environment
* Green – the new version of the application

---> You deploy your new version to Green while users are still using Blue.
Once the new version is fully tested and confirmed to be stable, you switch the traffic from Blue → Green. If something goes wrong, you can instantly roll back by routing traffic back to Blue.

---> Canary Deployment is a deployment strategy where you release a new version of your application to a small percentage of users first, while the majority continue using the old version. This helps you test the new version in real production with minimal risk. If everything looks good → slowly increase traffic to the new version. If something breaks → rollback immediately.
